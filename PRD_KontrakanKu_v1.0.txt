1. Overview & Tujuan Produk

1.1 Latar Belakang
KontrakanKu adalah platform digital terintegrasi berbasis Web dan Mobile Application yang dirancang khusus untuk memudahkan pengelolaan dan pemesanan properti kontrakan yang tersebar di berbagai lokasi berbeda. Platform ini menghubungkan pemilik kontrakan (admin/owner) dengan calon penyewa (user) secara efisien dan transparan.
Tantangan utama yang diselesaikan oleh platform ini adalah sulitnya mengelola portofolio kontrakan multi-lokasi secara terpusat, keterbatasan informasi bagi calon penyewa dalam menemukan kontrakan yang tepat, serta proses pemesanan dan pembayaran yang masih manual dan tidak efisien.

1.2 Tujuan Produk
Menyediakan tampilan peta interaktif yang menampilkan semua lokasi kontrakan secara real-time
Memberikan informasi lengkap setiap unit kontrakan (foto, kondisi, rating, ketersediaan)
Memfasilitasi proses pemesanan (booking) dan pembayaran secara online end-to-end
Memberikan notifikasi real-time kepada user dan admin terkait status transaksi
Menyediakan dashboard administrasi terpusat untuk monitoring dan pengelolaan semua properti
Menyediakan profil user lengkap dengan riwayat sewa dan pembayaran

5. User Flow & Journey

5.1 Flow User (Penyewa)
User buka app → Lihat peta dengan semua marker kontrakan
Klik marker → Popup info dasar muncul
Klik 'Lihat Detail' → Halaman detail kontrakan terbuka
Lihat foto galeri, rating, fasilitas, dan daftar unit tersedia
Pilih unit yang diinginkan → Klik 'Booking Sekarang'
Jika belum login → Redirect ke halaman Login/Register
Isi form booking: data diri, KTP, tanggal mulai, durasi sewa
Submit booking → Status: PENDING
Admin menerima notifikasi booking baru dan melakukan konfirmasi
User mendapat notifikasi: booking dikonfirmasi → Status: CONFIRMED
User diarahkan ke halaman pembayaran → Pilih metode bayar
Selesaikan pembayaran via payment gateway
Sistem verifikasi pembayaran → Status Booking: ACTIVE
User & Admin mendapat notifikasi konfirmasi pembayaran berhasil

5.2 Flow Admin
Admin login ke dashboard (web/mobile)
Lihat ringkasan di dashboard: unit kosong, pembayaran jatuh tempo, booking pending
Terima notifikasi booking baru → Review data penyewa & KTP → Konfirmasi atau Tolak
Monitor pembayaran: cek status, konfirmasi transfer manual jika ada
Kelola properti: tambah/edit unit, upload foto baru, update harga
Lihat laporan keuangan per properti per bulan

10. Open Questions & Asumsi

10.1 Open Questions
Apakah akan menggunakan Google Maps API (berbayar) atau Leaflet.js + OpenStreetMap (gratis)? Pertimbangkan biaya API key Google Maps untuk traffic tinggi.
Apakah mobile app dibangun sebagai React Native terpisah atau menggunakan pendekatan Capacitor (wrap React web)? Capacitor lebih cepat tapi fitur native lebih terbatas.
Apakah perlu fitur chat/messaging antara calon penyewa dan admin langsung di app?
Berapa lama data booking dan pembayaran perlu disimpan? (Compliance & audit trail)
Apakah sistem harga bersifat tetap per unit, atau perlu support harga fleksibel (harian, mingguan, bulanan)?
Apakah dibutuhkan fitur multi-bahasa (Bahasa Indonesia + English)?

10.2 Asumsi
Semua kontrakan diasumsikan berlokasi di Indonesia, menggunakan Rupiah sebagai mata uang transaksi.
Admin memiliki koneksi internet yang stabil untuk operasional sehari-hari.
User telah memiliki rekening bank atau e-wallet yang terdaftar untuk proses pembayaran.
VPS yang digunakan untuk Dokploy memiliki minimal spesifikasi: 4 vCPU, 8GB RAM, 100GB SSD.
PostgreSQL versi 15+ akan digunakan untuk memanfaatkan fitur terbaru termasuk dukungan JSON yang lebih baik.
